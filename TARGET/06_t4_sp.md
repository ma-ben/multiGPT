# T3. DP Bucket 证据化

把 reducer、bucket、grad sync 做成可调度对象，而不是默认黑盒。

**建议窗口：D14-D17**

| 子维度       | 任务                                                                                                                                                                                   |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Impl      | bucket manager 配置化；支持 bucket size sweep、参数注册顺序策略、main_grad view 校验                                                                                                                   |
| Comm/NCCL | 显式打印每个 bucket 包含哪些参数、大小、就绪顺序；验证 collective 是否按预期 group 触发                                                                                                                            |
| Overlap   | 做 4/16/25/64MB bucket sweep；再做“手动最后统一 all-reduce”的对照组，证明 reducer overlap 的价值                                                                                                         |
| Trace     | 至少两类图：bucket ready timeline、all-reduce overlap timeline                                                                                                                              |
| Recovery  | 梯度累计 + DP bucket 回归；保证不同 bucket 配置不影响 loss correctness                                                                                                                               |
| Source    | 读：PyTorch DDP reducer/bucket 文档与实现入口；你仓库 bucket 代码                                                                                                                                   |
| Failure   | 复盘 1 个 reducer 类 bug：bucket 过大拖晚触发、过小 host 开销高、参数顺序导致 overlap 差                                                                                                                      |
| 交付        | `reports/t3_dp_bucket.md`、bucket 组成清单、trace 对照                                                                                                                                       |
| 通过标准      | 你能把“bucket 边界 = overlap 粒度”讲成时间线而不是定义                                                                                                                                                |
