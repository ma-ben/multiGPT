# T4. Checkpoint 契约

不是直接上 DCP，而是先把恢复边界定义正确。

**建议窗口：D18-D20**

| 子维度       | 任务                                                                                                                                 |
| --------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| Impl      | 接通现有 checkpoint 保存与恢复；显式保存 model/optimizer/RNG/step/tokens；理清 TP/PP/DP/CP 下谁负责保存、谁负责加载                                              |
| Comm/NCCL | 标记 checkpoint 时 rank 侧 barrier、CPU/GPU copy、I/O 行为；先把阻塞点看清楚                                                                        |
| Recovery  | 先完成同拓扑恢复，不追求异拓扑；保证 train loop 可以从 checkpoint 继续往前跑                                                                                 |
| Trace     | 给出 save/load timeline，对比“不开 checkpoint”和“同步保存”的 step 影响                                                                           |
| Source    | 读：`checkpoint.py`、PyTorch state dict / optimizer state 约定、RNG 保存恢复                                                                 |
| Failure   | 复盘 1 个恢复类问题：状态缺项、wrapper 下 raw model 边界不清、optimizer state 不一致                                                                      |
| 交付        | `reports/t4_checkpoint_contract.md`、恢复演示日志、state contract 文档                                                                      |
| 通过标准      | 你能回答“当前恢复体系到底保存了什么、没保存什么、为什么这还不是 DCP”                                                                                              |
