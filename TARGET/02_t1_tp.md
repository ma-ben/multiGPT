# T1. TP 深挖：从“切张量”升级到“可解释的 TP 通信和 async dgrad overlap”

## 建议窗口

D4–D6

## 子维度任务

### Impl
清理 Column/RowParallelLinear、vocab/embedding/lm_head 语义；修 weight tying；做一版统一 TP tensor layout 文档。

### Comm/NCCL
明确每层 forward/backward 用的是 all-gather / reduce-scatter / all-reduce 哪一种；写 rank-local shape 表；对每个 collective 打事件名。

### Overlap
真正打开 TP dgrad async 路径；要求 trace 看到 dgrad collective 与后续 GEMM 交叠；Megatron 的 TP overlap 要求 `tensor_model_parallel_size >= 2`，并通常配 SP。([NVIDIA Docs][4])

### Trace
做 TP overlap off/on、不同 hidden/head 配置下的 3 组对照；输出 comm time、compute time、idle gap。

### Recovery
TP 模式下 checkpoint/save/load 回归；确认 tied params、sharded params 恢复后仍共享或语义一致。

### Source
读：Megatron TP/SP/TP overlap 指南；PyTorch autograd Function/stream 相关代码入口。

### Failure
复盘 1 个 TP 类 bug：head 数不整除、gather output 误用、权重 tie 断裂、async 路径没真的走到。

### 交付
`reports/t1_tp_deep.md` + 3 份 trace + tensor layout 图。

### 通过标准
你能回答“TP 为什么快/为什么慢、通信发生在哪、overlap 是否真的发生、恢复后参数语义有没有被 silently 改掉”。
