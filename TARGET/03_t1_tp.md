# T0b. 观测与时间线

先把“证据系统”搭起来，再讨论任何并行优化。

**建议窗口：D4-D5**

| 子维度       | 任务                                                                                                                                    |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| Impl      | 在 `train_step`、TP 通信、PP P2P、DP bucket wait、checkpoint save/load 周围接 `record_function`                                                |
| Trace     | 接 `torch.profiler` 导出 Chrome/Perfetto trace；按 step、microbatch、stage、bucket 分段                                                         |
| Metrics   | 每步统一输出 step time、tokens/s、max mem、comm wait time；把 wandb 指标和本地日志字段对齐                                                                   |
| Layout    | 让 trace 与 rank 日志能互相对应：看到某个 rank 的某个通信段，能反查到组和 shape                                                                                    |
| Recovery  | 先不追求异拓扑恢复；只要求 profiler 和日志不破坏训练主路径                                                                                                     |
| Source    | 读：PyTorch profiler、`record_function`、当前训练循环与通信入口                                                                                         |
| Failure   | 复盘 1 个“看起来异步，实际被同步点吃掉”的问题                                                                                                               |
| 交付        | `tools/profile_run.sh`、`reports/t0b_timeline.md`、一份 baseline trace                                                                        |
| 通过标准      | 你能完整讲出一次 step 的 timeline，而且不是靠 print 想象                                                                                                 |
