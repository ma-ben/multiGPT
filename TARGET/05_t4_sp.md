# T4. SP：不是新任务，是 TP 的第一层扩展

## 建议窗口

D13–D15

## 子维度任务

### Impl
在 LN/dropout/residual 等路径上引入 sequence shard；要求和 TP 组合跑通。

### Comm/NCCL
写清楚 SP 引入的 activation all-gather / reduce-scatter 位置；Megatron 明确建议 TP 场景开启 SP。([NVIDIA Docs][1])

### Overlap
打开 TP comm overlap；验证有 SP 时通信是否进入 bulk/pipelined overlap。

### Trace
对比 TP-only vs TP+SP：memory、step time、TP comm 占比。

### Recovery
TP+SP 下 checkpoint/save/load 回归；序列分片状态正确恢复。

### Source
读：Megatron parallelism guide 中 TP/SP 部分。

### Failure
复盘 1 个 SP 问题：residual path layout 混乱、gather 时机错误、activation 维度错位。

`reports/t4_sp.md`、SP tensor layout 图。
